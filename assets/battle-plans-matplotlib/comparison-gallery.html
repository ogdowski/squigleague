Validation Process Explanation

Step 1: Reference Analysis

  • Open Wahapedia image for each battle plan
  • Identify: deployment zone type (long edge/corner/quadrant), position, size
  • Count objectives and note their approximate positions
  • Identify special features (ley lines, rotation arrows, terrain markers)

Step 2: Battle Plan Data Cross-Reference

  • Read battle plan data from squire/battle_plans.py AOS_BATTLE_PLANS
  • Verify deployment type matches image
  • Count objectives from data structure
  • Extract special rules (seasonal rotation, ley lines, corruption spreading)

Step 3: Matplotlib Generation

  • Create 60"×44" battlefield grid (standard AoS table)
  • Draw deployment zones matching reference positions
  • Place objectives at positions approximating Wahapedia layout
  • Add special features (arrows, connection lines, markers) where applicable
  • Use consistent color scheme: Blue/Red zones, Orange objectives, Green terrain

Step 4: Visual Validation

  • Side-by-side comparison (this gallery)
  • Verify deployment zones are in correct positions
  • Count objectives in matplotlib matches count in reference
  • Check objective spacing is proportionally similar
  • Confirm special features are represented

Step 5: Functional Testing

  • Test API endpoint with image_version=matplotlib
  • Verify images serve correctly in gallery
  • Check filenames match exactly (aos-battle-plan-name.png)
  • Confirm file sizes reasonable (50-70KB per image)

IP Safety: Matplotlib diagrams are 100% programmatically generated with no Games Workshop assets. They represent tactical information only (zones, objectives, measurements) which cannot be copyrighted. The visual style is generic military tactical map aesthetic, not GW's specific artwork.
